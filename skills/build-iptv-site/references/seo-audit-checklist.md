# SEO Audit Checklist

Step 07 runs `iptv-seo-auditor` against the built Astro `dist/` before deploy. This is the checklist the auditor follows — use it manually too if you're spot-checking.

## 1. Technical SEO

- [ ] **Unique `<title>`** per page, 50–60 chars, ends with ` · {brand_name}` or ` | {brand_name}`
- [ ] **Unique `<meta name="description">`** per page, 140–160 chars
- [ ] **Canonical URL** set on every page, matches the final Cloudflare URL (no staging domains)
- [ ] **`<html lang="{target_language}">`** attribute matches the site's single locale
- [ ] **No `hreflang`** tags — single locale only
- [ ] **`robots.txt`** allows all, explicitly references `/sitemap-index.xml`
- [ ] **`sitemap-index.xml`** generated by `@astrojs/sitemap`, lists every page
- [ ] **Security headers** (`_headers`): CSP, HSTS, X-Content-Type-Options, X-Frame-Options, Permissions-Policy
- [ ] **www→apex redirect** in `_redirects` returns 301, not 302

## 2. On-page SEO

- [ ] **H1 present exactly once** per page
- [ ] **H1 is not a literal keyword match** (e.g. primary_keyword = "abonnement iptv", H1 = "Chaînes sans limites en 4K")
- [ ] **Primary keyword** appears in H1, first 100 words, and at least one H2
- [ ] **No duplicate H1** across the site
- [ ] **Internal links** from homepage → pricing + free-trial + every device page
- [ ] **Internal links** from every device page → pricing + free-trial (above the fold)
- [ ] **Breadcrumb** trail on non-homepage pages

## 3. Schema.org (JSON-LD)

Validate with Google's Rich Results Test + schema.org validator.

- [ ] **Organization** injected on every page by `BaseLayout.astro`
- [ ] **WebSite** with `SearchAction` injected on every page
- [ ] **Product + Offer** per plan on `/pricing/`
- [ ] **HowTo** on every `/devices/{slug}/` page with named steps
- [ ] **FAQPage** on `/faq/` with all Q&A
- [ ] **Article** on every blog post
- [ ] **BreadcrumbList** on every non-homepage page
- [ ] **ContactPage** on `/contact/`
- [ ] **AboutPage** on `/about/`

## 4. Content quality

Run `tools/content_linter.py`:

- [ ] Every file has all frontmatter fields
- [ ] No banned phrases (see `banned-phrases-dmca.md`)
- [ ] Currency format matches locale (regex-checked)
- [ ] Language detection ≥99% confidence (`lingua-language-detector`)
- [ ] No duplicate paragraphs across pages
- [ ] Every numeric claim matches an entry in `verified_claims.json`

## 5. Performance (Lighthouse / PageSpeed Insights)

Run `tools/pagespeed.py --strategy mobile` and `--strategy desktop`:

- [ ] **Performance ≥ 90** mobile, ≥ 95 desktop
- [ ] **SEO = 100**
- [ ] **Accessibility ≥ 90**
- [ ] **Best Practices ≥ 90**
- [ ] **LCP** < 2.5s mobile
- [ ] **CLS** < 0.1
- [ ] **TBT** < 200ms mobile
- [ ] All **images WebP**, `loading="lazy"` except above-fold
- [ ] Fonts **preloaded** (`<link rel="preload" as="font">`) for Plus Jakarta Sans 400/500/600/700
- [ ] **No render-blocking JS** (use `is:inline` or defer)
- [ ] **`compressHTML: true`** in `astro.config.mjs`
- [ ] **`build.inlineStylesheets: 'auto'`** in `astro.config.mjs`

## 6. Accessibility

Run `tools/a11y_check.py` (Playwright + axe-core):

- [ ] Zero axe-core violations at serious or critical level
- [ ] All interactive elements reachable by keyboard
- [ ] `:focus-visible` ring on every interactive element
- [ ] Alt text on every `<img>` (empty for decorative)
- [ ] Color contrast: brand primary on white ≥ 4.5:1, brand secondary on white ≥ 4.5:1 (check `brand.secondary` — the IPTV Helder source palette needed darkening to pass AA)
- [ ] Mobile menu uses `aria-expanded` + `aria-controls`
- [ ] No positive `tabindex` values (except `0`/`-1`)
- [ ] Form labels associated with inputs (`for`/`id` or wrapped)

## 7. Link integrity

Run `tools/link_check.py`:

- [ ] Zero 404s on internal links
- [ ] All external links return 2xx or 3xx
- [ ] All blog images exist at their declared paths
- [ ] `checkout_url` in every `Plan` resolves to a valid checkout

## 8. Social meta

- [ ] **`og:title`** on every page
- [ ] **`og:description`** on every page
- [ ] **`og:image`** set, either `/images/og-default.webp` or page-specific
- [ ] **`og:type`** = `website` (homepage, trust pages) or `article` (blog posts)
- [ ] **`og:locale`** matches target language locale code (`nl_NL`, `de_DE`, etc.)
- [ ] **Twitter Card** = `summary_large_image`
- [ ] OG image dimensions ≥ 1200×630 and < 5 MB

## 9. Deploy readiness (step 08 preconditions)

- [ ] Build succeeds with zero warnings (`npm run build`)
- [ ] `dist/` size under 10 MB
- [ ] No `console.log` or debug output in shipped JS
- [ ] Favicons in place (svg, 32px png, 180px apple-touch-icon)
- [ ] `theme-color` meta matches brand primary hex

## 10. Post-deploy (step 08 follow-ups)

- [ ] Sitemap submitted to Google Search Console
- [ ] Bing Webmaster Tools submitted
- [ ] First round of URL Inspection API calls logged (`tools/check_indexed.py`)
- [ ] Cloudflare Web Analytics enabled (not Google Analytics — slower, flagged by trackers)
- [ ] HTTPS redirect verified (apex + www)
- [ ] 404 page exists and is discoverable
