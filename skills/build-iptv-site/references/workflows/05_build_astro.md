# Workflow: Build Astro Project (iptv-tech-builder)

**Goal:** Scaffold a production-ready Astro project for IPTV, wire in all content, components, schema, and configuration. Single-language, single-country — no i18n.

---

## Required Inputs
- `.tmp/{country}_{lang}/content/` (all page copy)
- `.tmp/{country}_{lang}/page_map.json`
- Brand inputs (name, domain, colors, design personality, pricing, payment methods)

---

## Step 1 — Scaffold

From the target project root:

```bash
npm create astro@latest . -- --template minimal --typescript strict --no-install --git false
npm install
npx astro add tailwind cloudflare sitemap --yes
npm install gsap @astrojs/image @astrojs/rss
npm install -D astro-robots-txt
```

Verify each command succeeds before the next.

---

## Step 2 — Astro config

`astro.config.mjs` must include:
- `site: 'https://{domain}'`
- `output: 'static'` (Cloudflare Pages)
- `adapter: cloudflare()`
- Sitemap integration **without hreflang** (no i18n)
- `compressHTML: true`
- `build.inlineStylesheets: 'auto'`

## Step 4 — Content collections schema

`src/content/config.ts` must define collections matching the frontmatter in `04_write_content.md`:

```ts
import { defineCollection, z } from 'astro:content';

const pages = defineCollection({
  type: 'content',
  schema: z.object({
    page_type: z.enum(['homepage','pricing','trial','channels','about','contact','faq']),
    path: z.string(),
    primary_keyword: z.string(),
    secondary_keywords: z.array(z.string()),
    meta_title: z.string().min(50).max(62),
    meta_description: z.string().min(140).max(162),
    h1: z.string(),
    og_image: z.string().default('/images/og-default.webp'),
    schema_types: z.array(z.string()),
  }),
});

const devices = defineCollection({ type: 'content', schema: /* device schema */ });
const blog = defineCollection({ type: 'content', schema: /* blog schema */ });
const faqs = defineCollection({ type: 'data', schema: /* faq schema */ });

export const collections = { pages, devices, blog, faqs };
```

Validation failures at build time = reject and fix upstream content, never relax schema.

## Step 5 — Schema.org JSON-LD

Build a single `SchemaJsonLd.astro` component that accepts a `types` prop and emits the right combined schema per page type.

Required schema per page type:

| Page | Schema types |
|---|---|
| Homepage | `Organization`, `WebSite` (with SearchAction), `BreadcrumbList` |
| `/pricing/` | `Product` + `Offer` per plan, `BreadcrumbList` |
| Device page | `HowTo`, `BreadcrumbList` |
| `/faq/` | `FAQPage`, `BreadcrumbList` |
| Blog post | `Article`, `BreadcrumbList` |
| `/about/` | `AboutPage`, `Organization`, `BreadcrumbList` |

`Organization` schema fields:
- `name`, `url`, `logo`, `sameAs` (social handles), `contactPoint` (email, WhatsApp/Telegram)

Never emit schema for data not present on the page. Google flags this.

## Step 6 — Design personality → component treatment

The brand input `design_personality` takes one of four values. Apply the treatment below. Full details of which components are affected are in `agents/iptv-tech-builder.md`.

### Bold and modern
- **Shape language:** sharp clip-paths, diagonal section dividers, strong angular edges
- **Motion:** fast, heavy easing (power3.out), aggressive hover states
- **Transitions:** hard clip-path angles between sections
- **Shadows:** large brand-colored drop shadows
- **Color:** high contrast, saturated gradients

### Warm and approachable
- **Shape language:** rounded-3xl to rounded-4xl corners, wave section dividers, soft curves
- **Motion:** gentle, longer durations (0.9–1.2s), soft easing (power1.inOut)
- **Transitions:** wave/curve SVG dividers between sections
- **Shadows:** softer, warm-toned
- **Color:** warm gradient meshes behind hero and CTA

### Sleek and minimal
- **Shape language:** thin accent lines, minimal shapes, lots of whitespace
- **Motion:** subtle, short distances, quick durations (0.3–0.5s)
- **Transitions:** soft gradient fades between sections
- **Shadows:** restrained, single-layer
- **Color:** muted palette, section padding py-32+, whitespace-forward

### Energetic and dynamic
- **Shape language:** zigzag dividers, rotated elements, varied bento grid card sizes
- **Motion:** bouncy easing (back.out), playful rotation on scroll
- **Transitions:** zigzag or arrow section dividers
- **Shadows:** colorful, vibrant
- **Color:** vibrant gradient meshes, high color variety

## Step 7 — Required meta tags (BaseLayout.astro)

- `<title>` from content frontmatter
- `<meta name="description">`
- `<link rel="canonical">` — always the production URL, no query params
- Open Graph: `og:title`, `og:description`, `og:image`, `og:url`, `og:type`, `og:locale` (set to `{target_language}_{target_country}`)
- Twitter: `twitter:card=summary_large_image`
- `<meta name="robots" content="index,follow">`
- **No `hreflang` tags** (single locale)
- Favicon + apple-touch-icon
- Theme color matching primary brand color

---

## Step 8 — Performance budget

- Total page weight < 500 KB on homepage
- LCP < 2.0s, CLS < 0.05, INP < 150ms
- Inline critical CSS
- Preload hero image
- All images: WebP, width/height attributes, `loading="lazy"` except hero (`fetchpriority="high"`)
- GSAP imported only on pages that use it — never in BaseLayout
- No third-party analytics at build time (user adds post-launch)

---

## Step 9 — Robots & sitemap

`robots.txt`:
```
User-agent: *
Allow: /
Disallow: /api/

Sitemap: https://{domain}/sitemap-index.xml
```

Sitemap auto-generated by `@astrojs/sitemap`. Exclude `/legal/*` from sitemap priority boost.

---

## Step 10 — Build & verify

```bash
npm run build
```

Fail conditions (any = restart step, don't patch over):
- Build errors
- Missing images referenced in content
- Schema validation errors
- Any page missing canonical tag
- Any page with title/description out of char bounds
